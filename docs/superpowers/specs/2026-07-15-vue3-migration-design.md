# SEBO 官方网站 V6 Vue 3 迁移设计

## 目标

将 `02_完整工程源码` 中基于 React 19、Next/Vinext 与 Vite 的 SEBO 官方网站完整迁移为 Vue 3 项目，同时保留现有中英文内容、视觉表现、公开路由、SEO、离线浏览能力、Cloudflare Worker 表单接口，并兼容手机与 PC。

## 迁移范围

- 完整保留验收清单中的 208 条中英文公开路由及其 URL。
- 完整保留现有图片、文案、内容状态说明和品牌视觉。
- 保留首页、解决方案、技术与产品、来电啦服务、典型应用、新闻与洞察、案例、下载、关于、招聘、联系和 404 页面。
- 保留中英文切换、移动导航、应用筛选和联系表单交互。
- 保留 `/api/leads` Cloudflare Worker 接口、D1 数据结构、安全响应头、校验、限流和防机器人字段。
- 保留 sitemap、robots、页面元数据、分享图和静态离线交付。
- 不新增未经原站批准的产品规格、案例、下载资料或营销声明。

## 推荐架构

采用 Vue 3 + TypeScript + Vite + Vue Router。前端由 Vite 构建为静态资源，Cloudflare Worker 继续负责 `/api/leads` 和静态资源托管。这样既保持当前部署能力，也避免引入 Nuxt 的服务端运行约束。

项目按职责拆分：

- `src/app`：应用入口、路由、全局元数据和根布局。
- `src/components`：页头、页脚、按钮、卡片、页面首屏、联系横幅等公共组件。
- `src/pages`：按首页、解决方案、技术产品、服务、应用、内容、公司与联系等业务域组织页面。
- `src/data`：产品、应用、服务、新闻、导航和公开路由的类型化数据。
- `src/composables`：语言、路由链接和表单提交等共享逻辑。
- `src/styles`：设计变量、基础样式、布局、组件和响应式规则。
- `worker`、`db`、`drizzle`：保留现有后端职责，只调整与 Vue/Vite 构建衔接相关的部分。

## 路由与多语言

Vue Router 使用 HTML5 history。中文路由保持现有根路径，英文路由保持 `/en` 前缀。固定页面使用显式路由，产品、应用、服务、新闻和公司子页由类型化数据生成路由记录。未知地址进入 Vue 404 页面，并由托管配置回退到 `index.html`。

语言由当前 URL 决定，不依赖浏览器缓存。语言切换保持当前页面的对应路径；没有特殊映射时，仅增加或移除 `/en` 前缀。所有站内链接由统一辅助函数生成，避免中英文路径漂移。

## 页面与组件边界

根布局统一提供跳转主内容链接、页头、主内容和页脚。页面级组件只负责组合业务内容；重复 UI 拆为公共组件。产品、应用、服务和新闻详情通过路由参数查找类型化数据并渲染对应详情页。找不到数据时显示 404，而不是退回列表页。

React 的状态迁移为 Vue Composition API：移动菜单和联系类型使用 `ref`，应用筛选使用 `computed`，表单提交使用独立 composable。路由变化后关闭移动菜单并恢复合理的滚动位置。

## 响应式设计

延续现有 CSS 变量、字号比例、颜色、间距、图片裁切和视觉层级。桌面端保留居中导航和多列网格；移动端在不改变信息顺序的前提下切换为抽屉式导航、单列或双列卡片、可换行筛选和纵向表单。

主要验证视口为 1440×900 和 390×844，并补充 1920×1080、1024×768、768×1024、375×812。所有视口均要求无页面级横向滚动，导航、按钮和表单控件可触达，正文无需缩放阅读。遵循 `prefers-reduced-motion`，键盘焦点清晰可见。

## SEO 与静态输出

每个路由根据其页面类型和语言设置 title、description、canonical、Open Graph 与语言替代信息。构建阶段生成与公开路由清单一致的 `sitemap.xml` 和 `robots.txt`。静态离线交付阶段为全部公开路由生成可直接访问的目录入口，资源统一使用可离线解析的路径。

## 表单与错误处理

联系表单保留当前字段、必填约束、同意状态、来源路径、语言、蜜罐字段、提交中状态、成功编号和双语错误提示。前端只做体验型校验，Worker 继续作为安全和数据校验的权威边界。网络或服务错误不会清空用户输入；成功后才重置表单。

未知路由显示双语 404。图片保留替代文本策略，非内容性图片使用空替代文本。构建或数据校验发现重复路由、缺少对应语言路径或缺失资源时应直接失败。

## 验证与验收

- 单元测试覆盖语言路径转换、路由生成、应用筛选和表单负载构造。
- 路由测试验证 208 条公开路径全部可解析，未知路径进入 404，路由无重复。
- 构建测试验证 Vue 生产构建成功，不再包含 React、Next 或 Vinext 运行依赖。
- 静态检查验证公开页面引用的本地资源全部存在。
- 浏览器验收覆盖首页、技术产品、典型应用、联系页及至少一个中英文详情页。
- 在主要桌面与手机视口检查导航、布局、筛选、表单、焦点状态和横向溢出。
- Worker 接口使用本地测试验证成功、字段错误、蜜罐、请求体过大和限流路径。

## 完成标准

只有在 Vue 3 项目可安装、测试、构建，全部 208 条公开路由与资源检查通过，代表性页面在手机和 PC 浏览器验证通过，表单接口行为保持一致，且运行依赖中不再包含 React、React DOM、Next 或 Vinext 时，迁移才算完成。
